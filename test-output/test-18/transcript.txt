A Bloom filter is just a bit array combined with handful of hash functions. When you add a username, it's hashed multiple times and each hash sets a corresponding bit in the array. To check a username, you hash it the same way. If any of those bits is zero, the username is definitely not in the set. If all the bits are one, then it's probably present. And that's when you fall back to a more expensive check like a database query. What makes Bloom filters so powerful is that they never give false negatives. If they say the username isn't present, you can trust that. The only trade-off is the possibility of false positives, but that's usually acceptable when the alternative is scoring a massive database. To store 1 billion usernames with a 1% false positive rate, you would need roughly around 1.2 GB of memory.
