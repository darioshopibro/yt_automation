RAG stands for retrieval augmented generation. A user has a question, so they create a prompt. You'll need multiple data sources to answer a specific question, whether that's a PDF, a business application, or images.

We take this data and put it into a vector database. A vector database is a mathematical representation of structured and unstructured data, similar to arrays. These arrays are easier for machine learning models to understand versus just the raw unstructured data.

We query our vector database and get back an embedding with the relevant data for our prompt. That feeds back into the prompt. Once we're at this point, we move over to the large language model, which produces the output with the answer to our original question.
