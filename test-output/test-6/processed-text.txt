Here on git hub I have a node J S web app. In order to deliver this out to my customers, I need to run three commands: test, build, and deploy. I can automate this entire process in the cloud by using a C I service like git hub Actions. First I create a workflow, and then I tell it to run on every push to the master branch. The event triggers a job that runs on a Linux container in the cloud. We tell the container what to do as a series of steps. First it checks out the code in this git hub repo, then sets up node J S, installs my dependencies, and runs my test, build, and deploy commands. Now anytime we commit code to the master branch, it will run this workflow.
